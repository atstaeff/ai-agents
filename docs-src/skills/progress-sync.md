# Progress Sync

> Standup-Formate, Statusupdates und effektive Fortschrittskommunikation im Team.

## Standup-Format

```markdown
### Was habe ich gestern erreicht?
- Feature X implementiert und getestet

### Was plane ich heute?
- Integration Tests fuer Feature X
- Code Review fuer PR #42

### Gibt es Blocker?
- Warte auf API-Dokumentation vom Backend-Team
```

## Best Practices

- Kurz und praegnant (max. 2 Minuten pro Person)
- Blocker sofort eskalieren
- Async-Updates fuer verteilte Teams
- Visuelle Dashboards nutzen
